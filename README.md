## Issue の扱いについて

Discussions の内容が **開発タスクとして扱うべき** と判断された場合、  
開発側が Issue を作成し、進行状況を管理します。

### 開発側が Issue を作るケース
- 不具合として確認されたもの  
- 実装予定の機能要望  
- 長期的にトラッキングすべき項目  
- 明確な改善タスク
